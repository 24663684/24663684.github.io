IDataStore.RetrieveEnd(object sender, DwRetrieveEventArgs e) Method

.NET Standard 2.x | Current Version(0.5.0-alpha)

0.5.0-alpha

0.5.1-alpha (current)

在DataStore完成数据检索时执行该方法。该方法默认触发OnRetrieveEnd事件。在DataStore的子类中可以重写该方法。

This method is called when the DataStore completes the data retrieval. It triggers the OnRetrieveEnd event by default, and it can be overridden in a child class of the DataStore.

Namespace: Appeon.Data.PowerBuilder

Assembly: Appeon.Data.PowerBuilder.dll

Syntax

  public void RetrieveEnd(object sender, DwRetrieveEventArgs e);

Parameters

sender System.Object

An event sender in which the event occurred.

It is a reference to the current DataStore by default.

e Appeon.Data.PowerBuilder.DwRetrieveEventArgs

A DwRetrieveEventArgs object which can be used to pass by event arguments.

Remarks

The number of rows retrieved in the DwRetrieveEventArgs.RowCount argument is an unfiltered value.

Examples

演示RetrieveEnd方法,在Retrieve完成后,获取Rowcount的值

using Appeon.Data.PowerBuilder;
using System;

namespace Appeon.ApiDoc.IDataStoreExamples
{
    public class RetrieveEndExample
    {
        private SchoolContext _context;

        public RetrieveEndExample(SchoolContext dataContext)
        {
            // Set Data Context
            _context = dataContext;
        }

        public void Example()
        {
            // Instantiate the datastore with datawindow:d_department
            var datastore = new DataStore("d_department", _context);

            //定义retrieve结束时，需执行的操作
            datastore.OnRetrieveEnd +=
                (sender, args) =>
                {
                    Console.WriteLine("After Retrieve, Rowcount:{0}", args.RowCount);
                };

            //Retrieves rows from the database for datastore
            datastore.Retrieve();

            /*This code produces the following output:
            
            After Retrieve, Rowcount:4           
            */

        }
    }
}
