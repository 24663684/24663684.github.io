IModelStore<TModel>.SaveChanges(DataContext context, UpdateSqlStrategy updateSqlStrategy) Method

.NET Standard 2.x | Current Version (0.5.0-alpha)

0.5.0-alpha

0.5.1-alpha (current)

Updates the data to the database, based on the state of the model objects being tracked and the state of the properties in each model object. At the same time, you can specify the SQL generation strategy when performing the update operation.

Namespace: Appeon.Data

Assembly: Appeon.Data.dll

Syntax

  public IDbResult SaveChanges(DataContext context, UpdateSqlStrategy updateSqlStrategy, bool batchExecute = false);

Parameters

context Appeon.Data.DataContext

A DataContext object which contains the database connection information.

updateSqlStrategy Appeon.Data.UpdateSqlStrategy

The SQL strategy when performing the update operation. Use UpdateSqlStrategy.Update to generate a SQL UPDATE statement or use UpdateSqlStrategy.DeleteThenInsert to generate a DELETE statement and an INSERT statement.

If this parameter is not used, it uses UpdateSqlStrategy.Update internally.

Remarks

Do not use the SaveChanges method, when ModelStore is not being tracked.

How to process the tracked info for each model object

ModelStore does nothing for the model object whose state is ModelState.New or ModelState.NotModified.

When the state of the model object is ModelState.NewModified, the SQL INSERT statement is generated according to the definition in TModel. When the state of the model object is ModelState.Deleted, the SQL DELETE statement is generated according to the definition in TModel. When the state of the model object is ModelState.Modified, the SQL UPDATE statement will be generated according to the definition in TModel (or SQL DELETE and SQL INSERT if the updateSqlStrategy argument is specified to UpdateSqlStrategy.DeleteThenInsert).

Database Transaction

If you have not used the context argument (a DataContext object) to start the transaction using the DataContext.BeginTransaction method, the SaveChanges method will automatically start a transaction internally. If the database is updated successfully, the database transaction will be committed, otherwise the database transaction will be rolled back.

Examples

The following code example demonstrates how to use the SaveChanges(DataContext, UpdateSqlStrategy) method to specify the SQL generation strategy when performing the update operation.

using Appeon.Data;
using Appeon.ApiDoc.Models.School;
using System;

namespace Appeon.ApiDoc.IModelStoreExamples
{
    public class SaveChangesExample
    {
        private readonly SchoolContext _context;

        public SaveChangesExample(SchoolContext dataContext)
        {
            // Sets the data context
            _context = dataContext;
        }

        public void Example2()
        {
            var courses =
                new ModelStore<Course>(ChangeTrackingStrategy.PropertyState);

            // Retrieves data.
            courses.Retrieve(_context);

            // Modifies the title.
            courses.SetValue(0, "Title", "New Titile 1");

            // It uses the UpdateSqlStrategy.Update strategy to generate SQL 
            // UPDATE statement for the update operation by default. 
            // You can omit UpdateSqlStrategy.Update. 
            var dbResult = courses.SaveChanges(_context, UpdateSqlStrategy.Update);

            Console.WriteLine("{0} record has been modified.",
                  dbResult.ModifiedCount);

            var onlineCourses =
               new ModelStore<OnlineCourse>(ChangeTrackingStrategy.PropertyState);

            onlineCourses.Retrieve(_context);

            // Modifies the URL at index 0.
            onlineCourses.SetValue(0, "URL", "http://NewURL");

            // It uses the UpdateSqlStrategy.DeleteThenInsert strategy to  
            // generate a SQL DELETE statement and a SQL INSERT statement for 
            // the update operation.
            dbResult = onlineCourses.SaveChanges(_context,
                UpdateSqlStrategy.DeleteThenInsert);

            Console.WriteLine();
            Console.WriteLine("{0} record has been deleted.",
                dbResult.DeletedCount);
            Console.WriteLine("{0} record has been inserted.",
                dbResult.InsertedCount);
            Console.WriteLine("{0} record has been modified.",
                  dbResult.ModifiedCount);

            /* This code example produces the following output:
            
            1 record has been modified.

            1 record has been deleted.
            1 record has been inserted.
            0 record has been modified.
            */
        }
    }
}
