ModelStore<TModel>.ModelStore( IEnumerable<IModelEntry<TModel>>modelEntries, ChangeTrackingStrategy strategy = ChangeTrackingStrategy.NotTrack**) Constructor**

Initializes a new instance of the ModelStore class using an IEnumerable<IModelEntry<TModel>> object. You can specify the change tracking strategy. Data changes are not tracked by default.

Namespace: Appeon.Data

Assembly: Appeon.Data.dll

Syntax

  public ModelStore(IEnumerable<IModelEntry<TModel>> modelEntries, ChangeTrackingStrategy strategy = ChangeTrackingStrategy.NotTrack);

Parameters

modelEntries System.Collections.Generic.IEnumerable<IModelEntry<TModel>>

A sequence of IModelEntry objects which contain data and state.

strategy Appeon.Data.ChangeTrackingStrategy

(Optional) A value of the ChangeTrackingStrategy enumerated datatype. The default is ChangeTrackingStrategy.NotTrack.

Examples

The following code example initializes a ModelStore object, specifies the tracking strategy, and inserts the model entry to ModelStore.

using Appeon.ApiDoc.Models.School;
using Appeon.Data;
using System;
using System.Collections.Generic;

namespace Appeon.ApiDoc.ModelStoreExamples
{
    public class ModelStoreExample
    {
        private SchoolContext _context;

        public ModelStoreExample(SchoolContext dataContext)
        {
            // Sets the data context
            _context = dataContext;
        }

        public void Example4(IEnumerable<IModelEntry<Person>> newStudents)
        {           

            // Initializes a ModelStore object
            // Specifies the tracking strategy to PropertyState
            // Inserts data to ModelStore
            IModelStore<Person> model = new ModelStore<Person>(newStudents,
                                ChangeTrackingStrategy.PropertyState);

            // Shows how many rows in ModelStore
            Console.WriteLine("Item Count = {0}.",
                model.Count);

            /*This code produces the following output:

            Item Count = 2.
           */
        }
    }
}
