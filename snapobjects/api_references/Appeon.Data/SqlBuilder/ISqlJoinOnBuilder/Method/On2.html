ISqlJoinOnBuilder.On(string left, SqlBinaryOperator sqlOperator, string right) 方法

.NET Standard 2.x | 当前版本(0.5.0-alpha)

0.5.0-alpha

0.5.1-alpha(current)

创建ON子句, 向ON子句添加搜索条件，在运算符左侧和右侧指定SQL表达式。

Creates an ON clause and adds a search condition to it. Specifies SQL expressions on the left and right of the operator.

命名空间： Appeon.Data

程序集： Appeon.Data.dll

语法

  ISqlJoinOnAndOr On(string left, SqlBinaryOperator sqlOperator, string right);

Parameters

left System.String

A SQL expression on the left of the operator.

SqlBinaryOperator Appeon.Data.SqlBinaryOperator

An enumeration value of SqlBinaryOperator, which is the operator to test the two expressions on the left and right.

right System.String

A SQL expression on the right of the operator.

Returns

Appeon.Data.ISqlJoinOnAndOr

Returns the ISqlJoinOnAndOr object which can be used to add more search conditions to the current ON clause.

Examples

一表连接另外一个表时，使用On设置连接条件

using Appeon.Data;
using System;

namespace Appeon.ApiDoc.ISqlJoinOnBuilderExamples
{
    public class OnExample
    {
        private readonly SchoolContext _context;

        public OnExample(SchoolContext dataContext)
        {
            // Set data context
            _context = dataContext;
        }

        public void Example2()
        {
            var sqlQueryBuilder = new SqlQueryBuilder();

            // 获取Department表中DepartmentID，Name; Course表中Title数据，
            // Department和Course表关联条件为Department.DepartmentID = Course.DepartmentID
            sqlQueryBuilder
                .Select("Department.DepartmentID")
                .Select("Department.Name")
                .Select("Course.Title")
                .From("Department")
                .Join("Course")
                .On("Department.DepartmentID",
                    SqlBinaryOperator.Equals,
                    "Course.DepartmentID");

            // 将sqlQueryBuilder语法转换成context对应数据库的可执行语句
            string sql = sqlQueryBuilder.ToSqlString(_context);

            Console.WriteLine(sql);

            // execute sql
            var departments = _context.SqlExecutor.Select<DynamicModel>(sql);

            /*This code example produces the following output:           
              SELECT
                [Department].[DepartmentID],
                [Department].[Name],
                [Course].[Title]
              FROM [Department] JOIN [Course]
              ON [Department].[DepartmentID] = [Course].[DepartmentID]
            */
        }
    }
}
