Model

Model is the short for the Data Model class or object which describes the application data entry, such as departments, courses, orders or customers.

Firstly, it can maps the application data entry to a database entry (table, view, stored procedure or a SQL query) by applying attributes to the class or properties.

E.g.: For query result set from database, you can apply TableAttribute or FromTableAttribute to the class to define to table it mapped to, apply SqlWhereAttribute to the class to define the search condition for the SQL SELECT statement, uses SqlParameterAttribute to make the SQL statement parameterized. For saving data changes to database, you can apply KeyAttribute to one or more properties in the class to defines the Primary key in database, or apply PropertySaveAttribute to defines how to save to property to database.

Secondly, it can define the relationship between one application data entry to another.

E.g.: When there is master-detail relationship which the department and courses. You can apply EmbededModelAttribute to a property, which represents the courses, in the department class.

Thirdly, after the model class is instantiated, a model object can represent a row of the application data entry, such as a department, a course or a customer.
