Remarks

当您的应用程序完成DataStore中的某些行并且您不希望更新影响数据库中的行时，请使用RowsDiscard。例如，您可以丢弃删除缓冲区中的行，这可以防止在调用Update时删除行。 使用“重置”清除DataStore控件中的所有行。 仅当更改当前行的行号时，RowsDiscard Method才会触发RowFocusChanging和RowFocusChanged Event。当前行只是一个数字，表示哪一行是当前行。如果当前行的编号保持不变，则行内容的更改不会触发Event。 假设您有一个包含两行的DataStore。如果当前行是第1行而RowsDiscard丢弃第1行，则第2行成为当前行，但其行号也从2更改为1.由于当前行号仍为第1行，因此不会触发Event。如果当前行是第2行，RowsDiscard丢弃第1行，因为当前行号已从第2行更改为第1行，因此会触发Event。
