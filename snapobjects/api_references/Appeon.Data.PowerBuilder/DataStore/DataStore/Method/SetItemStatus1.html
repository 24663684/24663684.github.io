Remarks

将数据检索到DataStore时，所有行和列的状态最初都为NotModified。在特定行的列中更改数据之后，要么是因为用户更改了数据，要么是以编程方式更改了数据，例如通过SetItem Method，该列的列状态将更改为DataModified。一旦检索到的行中任何列的状态更改为DataModified，行状态也将更改为DataModified。 当一行插入DataStore时，它最初的行状态为New,该行中的所有列最初的列状态为NotModified。在行中的列中更改数据后，要么是因为用户更改了数据，要么是以编程方式更改了数据，例如通过SetItem Method，列状态更改为DataModified。一旦插入行中任何列的状态更改为DataModified，行状态将更改为NewModified。当DataStore列具有默认值时，列的状态不会更改为DataModified，直到用户对该行中的列进行至少一次实际更改。 调用Update时，行的状态标志确定Update Method用于更新数据库的SQL命令。调用INSERT或UPDATE，具体取决于以下行状态： 1、NewModified，使用INSERT。 2、DataModified，使用UPDATE。

仅当满足以下两个条件时，列才包含在UPDATE语句中。

1、该列位于DataWindow对象维护的可更新列列表中。 2、该列的列状态为DataModified。

DataStore控件包括它生成的INSERT语句中的所有列。如果列没有值，则DataStore会尝试插入null。如果数据库不允许该列中的空值，则会导致数据库错误。
