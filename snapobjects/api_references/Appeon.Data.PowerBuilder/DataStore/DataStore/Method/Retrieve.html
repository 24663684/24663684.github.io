Remarks

检索行后，将应用DataStore对象的过滤器。因此，任何不符合筛选条件的检索行都会立即移动到筛选缓冲区，并且不包含在返回计数中。在为DataStore检索行之前，必须使用SetTransObject或SetTrans指定事务对象。如果使用SetTransObject，则还必须使用SQL CONNECT语句来建立数据库连接。通常，当您调用Retrieve时，DataStore中已有的所有行都将被丢弃并替换为检索到的行。您可以在RetrieveStartEvent中返回代码2以防止这种情况发生。在这种情况下，Retrieve会将任何检索到的行添加到缓冲区中已存在的行中。
