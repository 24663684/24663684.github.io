使用一个TransactionScope实例(具有指定的范围和COM +互操作性要求以及事务选项)为Controller中的Action方法自动管理事务。

Uses a TransactionScope instance (with the specified scope and COM+ interoperability requirements, and transaction options) to automatically manage transactions for Action methods in Controller.

Namespace: Appeon.Data.ClientSyncCore

Assembly: Appeon.Data.ClientSyncCore.dll

Syntax

public static MvcOptions UseTransactionScope(this MvcOptions options, TransactionScopeOption scopeOption, TransactionOptions transactionOptions, EnterpriseServicesInteropOption interopOption)；

Parameters

options Microsoft.AspNetCore.Mvc.MvcOptions

The MvcOptions object which provides programmatic configuration for the MVC framework.

scopeOption System.Transactions.TransactionScopeOption

An instance of the TransactionScopeOption enumeration that describes the transaction requirements associated with this transaction scope.

transactionOptions System.Transactions.TransactionOptions

A TransactionOptions structure that describes the transaction options to use if a new transaction is created. If an existing transaction is used, the timeout value in this parameter applies to the transaction scope. If that time expires before the scope is disposed, the transaction is aborted.

interopOption System.Transactions.EnterpriseServicesInteropOption

An instance of the EnterpriseServicesInteropOption enumeration that specifies how distributed transactions interact with COM+ transactions.

Returns

Microsoft.AspNetCore.Mvc.MvcOptions

Returns the MvcOptions object which provides programmatic configuration for the MVC framework.

Remarks

It creates a TransactionScope instance to start a transaction before executing each Action method in the Controller, and commit the transaction after executing the Action method successfully. If an exception has been thrown, the transaction is rolled back.
