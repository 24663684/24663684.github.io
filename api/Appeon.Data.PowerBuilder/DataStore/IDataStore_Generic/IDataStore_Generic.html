IDataStore接口代表一个DataStore对象。DataStore是一个基于.NET Standard开发的数据操作组件，是对PowerScript的DataStore对象向C#的迁移，提供了与PowerScript的DataStore基本一致的行为，并且扩展了符合C#语法特性的方法，比如Linq，委托，事件都在DataStore中得到很好的应用，与ASP.NET服务框架充分整合能够很好的与不同客户端进行数据交互，使用DataStore组件可以保持PowerScript的编码风格在服务端简单方便的进行数据操作。

The IDataStore interface represents a C# DataStore object. C# DataStore is a .NET version of the native PowerBuilder DataStore. It provides almost the same set of APIs as the native PowerBuilder DataStore, and allows you to use standard C# features such as LINQ, delegates and transactions.

You can migrate existing PowerBuilder DataWindow/DataStore data objects to become C# DataStore, and take advantages of the simple and convenient PowerScript coding styles to implement data operations at the server side.

C# DataStore can exchange data easily with different client types because it fully integrates with ASP.NET framework. C# DataStore is a totally different solution from the legacy PB.NET DataStore: It is fully optimized for handling large-scale and high concurrency access requests, and can yield overall significantly better performance and scalability.

Remarks

说明
DataStore is used in almost exactly the same way as the PowerBuilder DataStore; and on top of that, DataStore also supports to use the advanced .NET development methods, such as LINQ set operations. DataStore is very useful when you want to access the database and manipulate the data. 几乎一样的使用方式，集合并在此基础上扩充了一些.Net开发使用的高级方法，甚至可以像集合操作那样操作DataStore。 当你要访问数据并对数据进行操作时DataStore非常有用，你可以使用DataStore做以下事情:
1 访问数据库数据。Access the data in the database.
2 对数据进行增删改查并可以简单方便的把修改反映到数据库。Append, delete, modify or query the data and commit the change to the database conveniently.
3 可以对数据进行分组排序等。Group or sort the data.
4 自动管理数据库事物也可以进行手动管理。Manage the database transactions automatically or manually.
5 提供了对数据合法性验证。Check the data validity and integrity.
6 可以跟踪数据的修改。Track the data modification.
7 为PowerBuilder项目的迁移提供了较好的解决方案。Used as the replacement for the migrated PowerBuilder DataStore.

创建DataStore时必需关联到一个DataWindow对象，使用Retrieve方法对数据进行检索，并把数据放到Primary缓冲区，可以传入DataWindow中定义的参数进行数据检索，使用AddRow向DataStore中追加数据，DeleteRow可以删除DataStore的数据，但数据并没真正从DataStore中删除而是把数据移到Delete缓冲区，你也以可以对Delete缓冲区的数据进行操作,当需要排序时使用SetSort设置排序条件，调用Sort后会对数据进行排序，SetFilter可以设置你想要过滤的数据，被过滤的数据会被移到Filter缓冲区，update方法可以将修改保存到数据库。When DataStore is created, it must be associated with a DataWindow object. It uses the Retrieve method to obtain the data and place the data to the primary buffer. Parameters defined in the DataWindow can also be passed in to retrieve data. AddRow method can be used to append data to the DataStore; and DeleteRow method can be used to remove data from the DataStore, however, data is not directly deleted from the DataStore, it is only moved to the delete buffer where you can still manipulate it later. SetSort method can be used to set the sorting criteria and Sort method to sort the data according to the predefined criteria. SetFilter method can be used to filter the data, and the data will be moved to the filter buffer. Update method can be used to finally commit the change to the database.
DataStore还提供了许多事件用于数据被变时调用。包括OnRowDeleting, OnRowDeleted, OnRetrieveStart, OnRetrieveEnd, OnUpdateStart, OnUpdateEnd。DataStore also provides many events that will occur on data change; these events include OnRowDeleting, OnRowDeleted, OnRetrieveStart, OnRetrieveEnd, OnUpdateStart, and OnUpdateEnd.
