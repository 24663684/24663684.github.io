Remarks

过滤器会导致检索所有行，然后应用过滤器。即使设置了“按需检索”选项，Filter Method也会在应用过滤器之前检索所有行。 过滤器使用DataStore的当前过滤条件。要更改筛选条件，请使用SetFilter Method。 SetFilter Method等效于在DataStore画家的Rows菜单上使用Filter命令。如果在调用Filter Method之前未调用SetFilter来分配或更改条件，则DataStore将默认使用对象定义中的条件。 当Retrieve Method检索DataStore的数据时，PowerBuilder会应用为DataStore对象定义的过滤器（如果有）。您只需在使用SetFilter更改过滤条件后调用过滤器，或者由于处理或用户输入而更改了数据。 过滤器对复合报表中的DataStore没有影响。

Filter causes all rows to be retrieved and then it applies the filter. Even when the Retrieve As Needed option is set, the Filter method retrieves all rows before applying the filter.

Filter uses the current filter criteria for the DataStore. To change the filter criteria, use the SetFilter method. The SetFilter method is equivalent to using the Filter command on the Rows menu of the DataStore painter. If you do not call SetFilter to assign or change criteria before calling the Filter method, the DataStore will default to use the criteria in the object definition.

When the Retrieve method retrieves data for the DataStore, PowerBuilder applies the filter that was defined for the DataStore object, if any. You only need to call Filter after you change the filter criteria with SetFilter or if the data has changed because of processing or user input.

Filter has no effect on the DataStore in a composite report.
