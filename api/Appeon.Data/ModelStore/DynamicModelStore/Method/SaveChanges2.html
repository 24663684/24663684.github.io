IModelStore<TModel>.SaveChanges(DataContext context, UpdateSqlStrategy updateSqlStrategy) Method

.NET Standard 2.x | Current Version(0.5.0-alpha)

0.5.0-alpha

0.5.1-alpha(current)

当ModelStore正在被追踪时，将根据要跟踪的model object的状态以及每个model object中的属性的状态把数据更新到指定的数据库。同时可以指定在执行Update操作时的SQL产生策略。

If the ModelStore is being tracked, the data is updated to the specified database, based on the state of the model objects being tracked and the state of the properties in each model object. At the same time, you can specify the SQL generated strategy when performing the Update operation.

Namespace: Appeon.Data

Assembly: Appeon.Data.dll

Syntax

  public IDbResult SaveChanges(DataContext context, UpdateSqlStrategy updateSqlStrategy, bool batchExecute = false);

Parameters

context Appeon.Data.DataContext

A DataContext object which contains connection information.

updateSqlStrategy Appeon.Data.UpdateSqlStrategy

The SQL strategy when performing the Update operation.

Uses UpdateSqlStrategy.Update to generate an SQL Update statement or UpdateSqlStrategy.DeleteThenInsert to generate a Delete statement and an Insert statement.

If this parameter is not used, it uses UpdateSqlStrategy.Update internally.

Remarks

When the ModelStore is not being tracked, do not use the SaveChanges method.

How to process the tracked info for each model object

ModelStore do nothing for the model object whose state is ModelState.New or ModelState.NotModified.

When the state of the model object is ModelState.NewModified, the SQL Insert statement is generated according to the definition in the TModel;

When the state of the model is ModelState.Deleted, the SQL Delete statement is generated according to the definition in the TModel.

When the state of the model is ModelState.Modified, the SQL Update statement will be generated according to the definition in the TModel (or SQL Delete and SQL Insert if the updateSqlStrategy argument is specified to UpdateSqlStrategy.DeleteThenInsert, See SaveChanges);

Database Transaction

If you having use the context argument (a DataContext object) to start the transaction using DataContext.BeginTransaction, the SaveChanges method will automatically start a transaction internally. If the database is updated successfully, the database transaction will be committed, otherwise the database transaction will be rollbacked.

Examples

保存Model数据, 并指定Model更新方式

using Appeon.Data;
using Appeon.ApiDoc.Models.School;
using System;

namespace Appeon.ApiDoc.IModelStoreExamples
{
    public class SaveChangesExample
    {
        private readonly SchoolContext _context;

        public SaveChangesExample(SchoolContext dataContext)
        {
            // Set data context
            _context = dataContext;
        }

        public void Example2()
        {
            IModelStore<Course> courses =
                new ModelStore<Course>(ChangeTrackingStrategy.PropertyState);

            // 检索出courses中的数据
            courses.Retrieve(_context);

            // 创建一条Course记录
            Course course = new Course()
            {
                CourseID = 2,
                Title = "test2",
                Credits = 1,
                DepartmentID = 1
            };

            // 将记录插入到第二行
            courses.Insert(1, course);

            //保存Model数据
            var dbResult = courses.SaveChanges(_context, UpdateSqlStrategy.Update);

            Console.WriteLine("{0} records have been Updated into the database.",
                              dbResult.AffectedCount);

            /* This code example produces the following output:
            
            1 records have been Updated into the database.
            */
        }
    }
}
